# GENESIS — AI Code Safety Pilot

## Purpose

The AI Code Safety Pilot is for developers, founders, and teams using AI coding tools who want to understand where AI-generated code can create hidden execution risk.

GENESIS is designed around a simple control point:

```text
AI-generated code
→ safety gate
→ ALLOW / REQUIRE_APPROVAL / BLOCK
→ execution
```

## Who This Is For

* Developers using Cursor, Copilot, Claude Code, or ChatGPT for coding
* Founders building AI-assisted engineering workflows
* Engineering leads who want safer AI coding adoption
* AI tooling builders who need deterministic pre-execution gates

## Pilot Outcomes

A pilot can include:

* review of your current AI coding workflow
* identification of risky AI-generated code patterns
* mapping of where a safety gate should sit
* examples of ALLOW / REQUIRE_APPROVAL / BLOCK policy rules
* recommendation for GitHub / CI / local workflow integration
* first safety-gate implementation plan

## Example Risks

GENESIS currently demonstrates public-safe examples such as:

* `module.exports = null`
* `if(false)` dead-code gates
* destructive replacement of working logic
