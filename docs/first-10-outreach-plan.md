# GENESIS — First 10 Outreach Plan

## Goal

Get 10 real developer conversations about AI-generated code risk.

Do not optimize for likes.
Optimize for conversations.

## Target Criteria

Look for people who:

- use Cursor, Copilot, Claude, ChatGPT, or other AI coding tools
- build with AI agents
- post about developer tools
- work in startups or small engineering teams
- complain about AI-generated code quality
- care about code review, CI, or software reliability

## Outreach Message

Use this short message:

Hey — I'm testing a small public demo called GENESIS AI Code Safety Gate.

It shows a simple idea:

AI-generated code change
→ safety gate
→ ALLOW / REQUIRE_APPROVAL / BLOCK

Example risk:
module.exports = null;

Would love your feedback:
https://github.com/aicodesafety/genesis-ai-code-safety-demo

Question:
Have you ever had AI-generated code break something unexpectedly?

## Success Criteria

Minimum success:
- 10 people contacted
- 3 replies
- 1 real pain signal

Strong success:
- 10 people contacted
- 5 replies
- 2 people ask for a demo
- 1 pilot candidate

## What Counts As Pain

Pain is real if they say:

- "This happened to me"
- "AI broke something in our code"
- "We need review controls"
- "This should run in CI"
- "I would try this"
- "Can you show me how it works?"

## What Does Not Count

Not enough:

- likes
- stars only
- "cool idea"
- generic praise
- no workflow pain
