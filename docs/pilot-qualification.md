# GENESIS — Pilot Qualification Checklist

## Goal

Identify whether a person or team is a real pilot candidate.

## Qualification Questions

1. Are they actively using AI coding tools?
2. Are they using AI-generated code in real projects?
3. Have they experienced broken code, risky code, or review friction?
4. Do they have a workflow where a safety gate could fit?
5. Would they try a CLI, GitHub App, CI check, or editor extension?
6. Do they have urgency?
7. Would they pay for this if it prevented one serious failure?

## Strong Pilot Signal

A strong pilot candidate says something like:

- "We use Cursor every day."
- "We already had AI-generated code break something."
- "We need this before merging PRs."
- "Can this run in CI?"
- "Can you test it on our stack?"
- "How soon can we try it?"

## Weak Pilot Signal

Weak signal:

- "Interesting"
- "Looks cool"
- "Maybe later"
- "I don't use AI coding tools"
- "We don't have this problem"

## Pilot Offer

Start with a focused discovery pilot.

Suggested pilot language:

I'm running a small early pilot for GENESIS.

The goal is to map where AI-generated code enters your workflow and design a first safety-gate layer around it.

The pilot includes:

- AI coding workflow audit
- risk pattern review
- recommended integration point
- first safety policy map
- follow-up roadmap

## Pricing Test

Use pricing carefully.

Option A:
Free discovery pilot for first 3–5 serious teams.

Option B:
Paid pilot:
$500–$1,500 for an initial workflow audit and safety-gate design.

Option C:
Team pilot:
$2,500–$5,000 if they want deeper integration planning.

## Decision

Mark each lead as:

- NOT FIT
- INTERESTED
- PAIN CONFIRMED
- PILOT CANDIDATE
- PILOT OFFER SENT
- CLOSED
- LOST
