# GENESIS — Public Feedback Review Checklist 107

## Purpose

A pre-launch checklist to run before any public feedback draft is published.

This checklist must pass completely before any draft is submitted or posted.

One checklist run per channel, per post.

---

## SECTION 1 — Authorization Check

- [ ] User has issued the explicit approval phrase for this channel
- [ ] Approval phrase matches exactly one of the approved phrases in docs/public-feedback-strategy-107.md
- [ ] No other channel is being published simultaneously without its own separate approval phrase

**Stop if:** no approval phrase has been issued.

---

## SECTION 2 — Voice and Tone Check

- [ ] Post opens with a real problem, not a product announcement
- [ ] No use of: "revolutionary", "game-changing", "10x", "autonomous enterprise platform"
- [ ] No use of: "We are excited to announce", "Introducing GENESIS", "The future of AI is here"
- [ ] Language is human and founder-led, not corporate
- [ ] Paragraphs are short — one idea per paragraph
- [ ] Post does not sound like a press release

**Stop if:** any prohibited phrase or pattern is present.

---

## SECTION 3 — Claims Check

- [ ] No claim that GENESIS prevents all AI bugs
- [ ] No claim that GENESIS is enterprise-ready
- [ ] No claim that GENESIS is production-grade
- [ ] No claim that GENESIS replaces tests or code review
- [ ] No claim that GENESIS supports every stack
- [ ] No claim of guaranteed protection
- [ ] No exaggerated adoption numbers
- [ ] All claims are limited to: small demo, early validation, ALLOW / REQUIRE_APPROVAL / BLOCK classification

**Stop if:** any prohibited claim is present.

---

## SECTION 4 — Link Check

- [ ] Demo link is correct: https://aicodesafety.com
- [ ] Public repo link is correct if used: https://github.com/aicodesafety/genesis-ai-code-safety-demo
- [ ] No more than two links in the post
- [ ] No private repo link
- [ ] No checkout or payment link
- [ ] No Lemon Squeezy link
- [ ] No affiliate link

**Stop if:** any incorrect or prohibited link is present.

---

## SECTION 5 — Payment Check

- [ ] No mention of price
- [ ] No mention of checkout
- [ ] No mention of Lemon Squeezy
- [ ] No Buy Now button or phrase
- [ ] No paid pilot pitch
- [ ] No payment link of any kind

**Stop if:** any payment reference is present.

---

## SECTION 6 — Private Core Check

- [ ] No description of private GENESIS internals
- [ ] No mention of private repo
- [ ] No implication that the public demo includes the private core
- [ ] If the system is described, the allowed phrase is used:
      "This is a public demo shell. The private GENESIS core is not included."

**Stop if:** any private core detail is exposed.

---

## SECTION 7 — Feedback Question Check

- [ ] Post ends with exactly one clear feedback question
- [ ] Question is about workflow placement or real workflow pain
- [ ] Question is not: "Would you buy this?", "Who wants access?", "Can you share this?"
- [ ] Question invites a genuine reply, not a conversion action

**Stop if:** no feedback question is present, or question is a sales ask.

---

## SECTION 8 — Platform Rules Check

### GitHub Discussion
- [ ] Posted to the correct repository discussion board
- [ ] Not posted as an issue
- [ ] Title is a genuine question, not a pitch

### LinkedIn
- [ ] Posted from personal founder profile, not a company page blast
- [ ] Not a repost of identical content from another channel

### X / Twitter
- [ ] Character limit respected per post
- [ ] No misleading thread framing
- [ ] No mass-tag of unrelated accounts

### Show HN
- [ ] Title starts with "Show HN:"
- [ ] Plain text, no hype in body
- [ ] Links are correct
- [ ] Not a resubmission of a recent identical post

### Reddit
- [ ] Posted to correct community
- [ ] Community rules reviewed for self-promotion policy
- [ ] Question-first approach used if community prohibits direct links
- [ ] Link included only where community rules allow

### Dev.to
- [ ] Article is a genuine technical build log
- [ ] Tags are accurate and relevant
- [ ] Not a duplicate of content posted elsewhere under the same title

### Medium
- [ ] Essay is long-form and substantive
- [ ] Not a copy-paste of a LinkedIn post
- [ ] Contact email included at end if appropriate

**Stop if:** platform-specific rules are violated.

---

## SECTION 9 — Anti-Spam Check

- [ ] This exact post has not been submitted to this channel before
- [ ] This post is not identical to content posted on another channel today
- [ ] No plan to DM commenters from this post
- [ ] No automated reply tool connected to this post
- [ ] No mass-repost plan

**Stop if:** any spam pattern is present.

---

## SECTION 10 — Final Human Approval

- [ ] User has read the full draft immediately before posting
- [ ] User confirms the draft accurately represents the current state of GENESIS
- [ ] User confirms no prohibited content is present
- [ ] User confirms they are posting manually, not through an automated tool

**Stop if:** user has not personally reviewed and confirmed the draft.

---

GHOST / WEAK_SIGNAL / PAIN_CONFIRMED / INTEGRATION_INTEREST / DEMO_REQUEST / PILOT_CANDIDATE / NOT_RELEVANT

---

## VALIDATION CRITERIA

After 7 days from first post:

Review all responses.

If result is zero PAIN_CONFIRMED or INTEGRATION_INTEREST:

- Review positioning
- Review channel selection
- Do not build new features yet

If result includes one or more PAIN_CONFIRMED:

- Document the pain description exactly as stated
- Prepare integration discovery based on the pain

If result includes one or more PILOT_CANDIDATE:

- Move to pilot qualification
- Route conversation to hello@aicodesafety.com

---

## Approval Boundary

This checklist does not authorize any post.

Authorization requires a separate explicit approval phrase from the user.

This checklist is a pre-flight check, not a launch command.
